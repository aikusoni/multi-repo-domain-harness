# Agent Instructions

## 필수 선행 조건

이 저장소에서 작업하는 모든 에이전트는 **작업 종류와 규모에 관계없이 다른 파일을 읽거나 수정하기 전에
루트 `INDEX.md` 전체를 반드시 읽고 숙지해야 한다.** 이 절차를 생략한 상태에서는 작업을 시작하지 않는다.

`INDEX.md`는 하네스 운영 규칙의 유일한 정본이다. 이 파일 또는 다른 에이전트 지침과 충돌하는 내용이
있으면 `INDEX.md`를 우선한다. 작업 도중 `INDEX.md`의 규칙 갱신 스탬프가 바뀌면 전체를 다시 읽은 뒤
새 규칙에 따라 작업을 계속한다.

새 작업·작업 단위 전환·중단 후 재개·커밋 또는 최종 보고 직전에는 `INDEX.md` 상단 스탬프를 다시
확인한다. 마지막으로 읽은 값과 다르면 전체를 다시 읽기 전까지 작업을 계속하지 않는다.

## 시작 절차

1. 루트 `INDEX.md` 전체를 읽고 상단의 규칙 갱신 스탬프를 확인한다.
2. 현재 작업 대상의 프로젝트 식별자를 결정하고 `PROJECTS.md`에서 확인한다.
3. 실행 자동화 관련 작업이면 `OPERATORS.md`에서 operator 식별자와 소유 프로젝트를 확인한다.
4. `ISSUES.md`에서 해당 프로젝트와 관련 operator의 활성 게이트를 확인한다.
5. `changed/`의 해당 주체 확인 로그와 `tasks/<project>/` 또는 `tasks/operator-<id>/`의 미완료 작업을 확인한다.
6. 현재 작업에 직접 필요한 `docs/`와 `operators/` 문서만 추가로 읽는다.

## 변경 원칙

- 기준 문서에 확정되지 않은 내용을 단정해서 쓰지 않는다.
- 검증하지 않은 내용은 `[미검증]`으로 표시한다.
- 기존 journal과 이벤트 피드는 수정하지 않고 append한다.
- 다른 프로젝트가 수행하거나 operator가 실행·수정할 일은 `requests/`에 작성한다.
- 교차 프로젝트에 영향을 주는 변경은 `changed/`에 기록한다.
- 커밋 전 전체 변경에서 비밀·개인정보·내부 경로·비공개 시스템 정보 등 공개 위험 정보를 검토한다.
- 공개 위험 정보가 없으면 완료한 변경을 자동 커밋한다.
- 위험하거나 공개 가능 여부가 불확실한 정보가 있으면 stage와 commit을 중단하고 사용자에게 한 번 확인한다.
- 자동 push는 하지 않는다. 사용자가 명시적으로 요청한 경우에만 수행한다.
